The `SignIn` object holds the state of the current sign-in and provides helper methods to navigate and complete the sign-in process. It is used to manage the sign-in lifecycle, including the first and second factor verification, and the creation of a new session.

## Properties

| Property                                                     | Type                              | Description                                                               |
| ------------------------------------------------------------ | --------------------------------- | ------------------------------------------------------------------------- |
| <a id="id"></a> `id?`                                        | `string`                          | The unique identifier of the resource.                                    |
| <a id="pathroot"></a> `pathRoot`                             | `string`                          | The root path of the resource.                                            |
| <a id="status"></a> `status`                                 | <code>null \| SignInStatus</code> | The current status of the sign-in.                                        |
| <a id="supportedidentifiers"></a> ~~`supportedIdentifiers`~~ | <code>SignInIdentifier[]</code>   | **Deprecated.** This attribute will be removed in the next major version. |
